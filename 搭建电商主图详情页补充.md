PART 01

为什么是护肤品，不是衣服也不是 3C
很多人上来就拿衣服测，翻车率极高。相信做过的朋友都知道。

衣服对面料垂坠感、人体比例、穿搭协调性这几样要求太高，目前这一代模型还撑不住。3C 数码也不太适合，接口、按键、屏幕这些细节模型还在乱画。

护肤品反而是相对最稳的品类。

几个原因。瓶身上的中英文标签需要精准渲染，这恰好是 GPT-Image-2 这一代最强的能力。玻璃瓶、磨砂罐、乳霜质地这些材质，模型已经处理得相当真。而且护肤品从白底主图、成分解析到使用前后对比、场景氛围，图型种类极其丰富，刚好能把一个完整链接需要的图跑全。

我挑了一个虚构的 SKU——50ml 神经酰胺修护面霜，磨砂白罐加银色按压泵头。虚构的目的是为了不踩任何品牌版权问题，但产品规格是按真实护肤品的写法来的。


PART 02

一个新品上架到底要哪些图
不是 3 张，也不是 5 张，淘宝天猫一个完整的护肤品链接，通常需要 12 张：

· 白底主图 ×1（搜索结果页的第一印象）

· 卖点副图 ×4（详情页前五张里除主图剩下的那 4 张）

· 场景氛围图 ×2（让消费者带入"这就是我的生活"）

· SKU 多规格卡 ×1（不同容量横排）

· 详情页长图 ×1（从上拉到下那张信息长图）

· 促销主视觉 ×1（618、双 11 大促用）

· 广告投放素材 ×2（信息流 + 搜索广告各一张）

加起来正好 12 张。

一般电商公司都有美工和建模，做过运营的朋友肯定知道做这些图片没有个两三天是做不出来的。

而现在，下面这一套用 GPT-Image-2 跑下来，30 分钟出完，API 费用大概 17 块人民币。当然初稿不等于成片，后面我会展开讲。


PART 03

写 prompt 这事，别从空白页开始
很多人第一反应是"我自己从零憋"。

别这么干。

GitHub 上已经有很多社区验证过的 prompt 库摆在那儿，我们去仓库找现成的就行。

又因为刚才那个仓库里面的东西太多，所以我们只需搜索中国电商就行了。而中国电商场景一般要做三件事。把产品描述换成自己的——英文 prompt 里的产品名、材质、颜色全部替换成自家的。把文字内容换成中文，用「」中文引号包裹，渲染准确率明显高一些。把比例改成平台要的——海外 prompt 很多是 16:9 横版，国内电商主图要 1:1，这个必须改。

GitHub 的 prompt 是骨架，肉是你自己往上填的。

写 prompt 自己心里有几条铁律，下面 12 张图都是这几条的具体落地：

· 3 层信息架构：核心承诺（≤15 字）+ 关键证据（2 到 3 个）+ 行动指令（≤8 字）

· 留白必须显式声明："留白至少 50%"——不写 AI 一定画满

· 否定清单不能省：每条结尾加 不要 logo、不要水印、不要装饰元素

· 颜色用 hex 不用形容词：写 #FFFFFF 不写"白底"，写 #D4AF37 不写"金色"

· 产品占比必须数字化：白底主图 35-40%，场景图 20-25%

· 批量出图优于反复调参：一次 n=5 出 5 个方向，让 CTR 选风格，别让你的审美选风格

下面 12 张图，每张配 1 段思路 + 1 段完整 prompt。截图就能用。

IMG · 01

白底主图

这张图是消费者在搜索结果页看到的第一眼，重要性不用我多说。

写法上有几个细节。背景写 #FFFFFF 不写"白底"，"白底"出来的图背景是淡灰的。产品占画面 38%，35-40% 是白底主图的黄金比例，太小显廉价，太大显拥挤。然后是 3/4 微侧角度，比正面对镜头更有立体感。最后一个容易忽略的——顶部要留 200×100 的空白区给平台价格叠加，不写产品会把整个画面填满。

IMG · 03

成分解析副图

成分解析是护肤品转化率最高的副图类型，没有之一。

4 种核心成分用细线连接到瓶身，像 Apple 的产品拆解图那种感觉。背景从纯白换成浅米色 #F5F1E8，跟前面几张白底图拉开视觉节奏——连续 5 张纯白，消费者眼睛会疲劳的。

PROMPT

生成一张 1:1 电商成分解析副图。  背景：浅米色 #F5F1E8，微妙的纸张纹理感。  中央：面霜瓶（磨砂白罐，银色泵头），占画面 25%。  围绕瓶身，4 个成分标注区域，用细线连接到瓶身对应位置： - 左上：「神经酰胺 NP」→ 细线 → 瓶身，小字「修护皮肤屏障」 - 右上：「玻尿酸钠」→ 细线 → 瓶身，小字「深层补水锁水」 - 左下：「积雪草提取物」→ 细线 → 瓶身，小字「舒缓褪红」 - 右下：「维生素 B5」→ 细线 → 瓶身，小字「促进修护」  每个标注区域包含：成分名（16pt 深咖 #3D2B1F 加粗）+ 功效（12pt 浅咖 #8B7355）。  顶部标题：「4 大核心成分」32pt 深咖加粗。  风格：Apple Style 产品拆解图，科学信息图美学。 字体：SF Pro Display。 no watermark, no extra text。

PART 05

踩过的几个坑，你大概率也会遇到
过程不是一帆风顺，这个大家应该可以理解。下面看下出错的地方：

PIT · 01

中文字笔画错

标题里"修护屏障"出来一看，"障"字的右耳旁少了一竖。这种错肉眼看一遍容易漏，必须放大到 200% 逐字核对。复杂笔画的字（"赢""鬱""餮"这种）能换简单同义字就换，95% 的中文准确率不是 100%。

PIT · 02

品牌色漂移

Prompt 里写了"金色"，8 张图出来 8 种金，柠檬金、玫瑰金、古铜金……什么都有。后来全换成 hex 码 #D4AF37，才稳定下来。颜色形容词在 prompt 里基本都不可靠，能用 hex 就用 hex。

PIT · 03

Logo 区被填满

你写"留空间放 logo"，模型经常当没看见。必须写"左上角 200×100 像素区域完全留白"，给出精确坐标才行。

这几个坑我都踩过，每条 prompt 里已经做了对应的防护。但你自己跑的时候，出图后还是要逐张检查，尤其是文字。


PART 06

聊聊成本和"省下的时间"
12 张图，GPT-Image-2 API 高质量模式 $0.20 一张，合计 17 块人民币左右，30 分钟出完。如果是订阅的Pro模式不要钱简直不要太香。

公司美工建模：两三天，而且中间要不断沟通交流。

17 块出来的是初稿，不是上架级成片。你要花时间逐张检查文字、核对笔画、微调布局，第一批出图的质量也不会都让你满意，有的得重出。我自己跑这一套的时候，挑一个最接近的，再微调 prompt 出第二轮。这才是真实的工作流。

所以这件事的本质，不是"AI 替你做设计"，而是"AI 把从 0 到 80 分的环节帮你干了"。你专注做 80 到 100 分的精修，把摄影师设计师那些标准化、重复性的活儿外包给模型。

以前 80% 的时间花在从 0 到 80，现在 80% 的时间花在从 80 到 100，这才是真正的效率提升。
